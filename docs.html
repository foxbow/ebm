EasyBookMarks Documentation

EasyBookMarks ebm is a little PHP scriptset that allows you to collect and maintain Weblinks on a webpage. The main goal was to have a list of bookmarks that were not attached to the browser that's being used but were accessible anywhere, especially after a fresh install.

Over the time a couple of features and changes made the simple page what it has become now.

Installation

Installing the package

Just unpack the archive somewhere in the subtree that can be accessed by the webserver.

tar -zxpf ebm-version.tar.gz

Additionally you may need to set the user and group accordingly so the webserver may access the files.

After this you need to create a settings.php file, just copy the supplied settings.ini and edit it accordingly. This file just contains the description of the database access and will not be overwritten upon updating.

Maintenance

After installing and opening the index page for the first time you will get an error message complaining that a table could not be read out. This message triggers the creation of the needed tables and will not appear again.

With setting up the basic tables, the admin user called ebm will be created. You should now press the Login link and log in as ebm with the password ebm. After that the Login links will be replaced with an Admin links. Following this you'll be transported to the settings page where you can customize the features, create new users, delete users and change passwords. You should first change the password for ebm.

The Admin Page

Once you log in as user ebm you have access to the Admin page where you find the following options.

  • Title
    This is the text that will be set as <title> tag for the page. It will also be the headline.
  • Default category
    The category that will be opened as default. If that category does not exist, no category will be opened.
  • Add/edit without login
    Allows anyone who visits the site to add and edit bookmarks on the page.
  • Shortcut button
    Shows a button labelled P after each entry which allows the logged in user to copy that link into his personal default category.
  • Kill button
    Shows a button after each entry which allows to delete the entry without needing to open the edit view.
  • New window button
    Shows a button that will open the link in a new browser window. In fact it uses the target="_blank" value.
  • Contact address
    The email address that will be shown in the footer. If this is left empty a default will be set.
  • Force login
    With this flag set, every user has to log in before seeing any page, even the public bookmarks.
  • Show searchfields
    The searchfields are two textfields, one serving as a shortcut to dict.leo.org, a great online german/english translator and another shortcut to google.de.

Update the package

Just unpack the package over the old installation. If you did not do any modifications beyond editing settings.php you're done after resetting the user and group attributes.

Frequently Asked Questions

  • What about subcategories?
    The current data model does not offer the background for subcategories. If Version 2 ever makes it into the light it will have subcategories.
  • Will you support database?
    Only if I get to play with it, else feel free to write your own interface. Look at the db directory to find a sceleton and the provided interfaces. It shouldn't be hard to create an own with that help.
  • The created HTML is not W3 conform!
    Right, it isn't. Mainly because Microsoft's Internet Explorer can't handle proper HTML *D'Oh!* Nested forms and tables will look awful when programmed after the definitions.
  • Why do I need cookies?
    The main reason is, that Dillo still does not know how to handle AUTH requests. Furthermore it's more handy to stay logged in for a month. If enough people bug me 'll add a switch that allows to change the behaviour back to Apache's .htaccess style authentication.
